# ゲーム風ポートフォリオ（PC向けWebサイト）

HTML / CSS / JavaScript を用いて制作した、インタラクティブなゲーム風ポートフォリオです。  
トップ画面をRPG風マップとして構成し、キャラクター操作によって各セクションへアクセスできるUI体験を実装しています。

---

## ■概要

本プロジェクトは、従来の静的なポートフォリオではなく、  
「操作によって情報へアクセスするインタラクティブUI」として設計しています。

ユーザーはキャラクターを操作し、マップ上の建物に接触またはクリックすることで各セクションへ遷移します。

---

## ■使用技術

- HTML5
- CSS3
- JavaScript（Vanilla JS）
- requestAnimationFrame
- DOM操作（absolute配置ベース）

---

## ■主な機能

### ● プレイヤー操作

- キーボード操作（WASD / 矢印キー対応）
- 2Dマップ上の自由移動
- 移動状態に応じたスプライトアニメーション

---

### ● アニメーション制御

- 待機・歩行モーションの切替
- フレーム制御によるスプライトアニメーション実装
- 移動状態に応じたリアルタイム更新

---

### ● 当たり判定

- hit-area方式による判定分離（見た目とロジックを分離）
- プレイヤーと建物の距離ベース判定

---

### ● ページ遷移

- 建物接触によるページ遷移
- 建物クリックによるページ遷移
  - About
  - Works
  - Skills
  - Contact
- フェードアウト演出による遷移表現

---

### ● UIインタラクション

- 建物ホバー時の視覚的フィードバック
  - カーソルに応じたインタラクション演出

---

## ■画面構成（Home）

ゲーム風マップとして以下の要素を配置しています。

- 家（About）
- 城（Works）
- 大樹（Skills）
- ポスト（Contact）
- プレイヤーキャラクター

---

## ■設計方針

- 静的UIではなく「操作可能なインタラクティブUI」を採用
- フロントエンド単体でゲーム的体験を実現
- 拡張性（NPC・イベント・BGM追加など）を考慮した構造設計
- UIとロジックを分離した実装

---

## ■工夫した点

- requestAnimationFrameによるゲームループ実装
- hit-area分離による当たり判定設計
- スプライトアニメーションによるキャラクター表現
- DOMベースでの軽量ゲーム的UI構築
- 建物接触＋クリック両対応の遷移設計
- ホバー時の視覚フィードバックによるUX向上
- 遷移時のフェードアウト演出実装
